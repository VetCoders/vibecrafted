## ؞Vibecraftedٜ — The Founders’ Framework that created itself.

### Opis

؞Vibecraftedٜ to framework dla AI-guided engineering. Nie jest narzędziem do generowania kodu ani zestawem promptów. To
system pracy z agentami, który prowadzi od pierwszego draftu do produktu gotowego do wypuszczenia.

Nie zakłada, że AI „pisze dobrze”. Zakłada, że kod generowany przez AI jest zmienny, często niekompletny i podatny na
błędy. Dlatego zamiast obiecywać inteligencję, wprowadza strukturę: kontekst, pamięć decyzji, analizę repozytorium i
iteracyjne pętle konwergencji.

Rezultat: proces, który systematycznie redukuje błędy i doprowadza projekt do stanu, w którym można go bezpiecznie
pokazać światu.

---

### Na czym polega różnica

Większość narzędzi kończy się na wygenerowaniu kodu.

؞Vibecraftedٜ zaczyna się tam, gdzie inne narzędzia kończą.

Agent nie zatrzymuje się na pierwszym rozwiązaniu. Każda zmiana jest:

- analizowana
- podważana
- poprawiana
- weryfikowana ponownie

Pętla trwa tak długo, aż przestają istnieć kontrprzykłady.

To nie jest obietnica jakości. To jest mechanizm dochodzenia do jakości.

---

### Model działania

Framework organizuje pracę w trzy fazy:

**Craft** — tworzenie rozwiązania  
Planowanie, scaffold, implementacja, delegowanie zadań agentom

**Converge** — redukcja błędów  
Follow-up, analiza kontrprzykładów, iteracyjne poprawki, cleanup

**Ship** — dowiezienie produktu  
Audyt gotowości, UX, packaging, dokumentacja, dystrybucja

Każda faza ma konkretne workflowy i narzędzia. Agent nie działa w próżni — działa w systemie.

---

### Warstwy systemu

| Warstwa          | Rola                                                     |
| ---------------- | -------------------------------------------------------- |
| **Percepcja**    | Analiza struktury repozytorium, zależności, wpływu zmian |
| **Kontekst**     | Pamięć decyzji i poprzednich iteracji                    |
| **Egzekucja**    | Implementacja i delegowanie zadań agentom                |
| **Konwergencja** | Eliminacja błędów i kontrprzykładów                      |
| **Weryfikacja**  | Audyt tego, co nadal nie jest gotowe                     |
| **Dostarczenie** | Packaging, UX, release                                   |

Agent nie zgaduje architektury. Czyta ją.  
Agent nie „zakłada że działa”. Musi to udowodnić.

---

### Dlaczego to działa

AI generuje kod stochastycznie.  
To oznacza, że bez struktury produkuje zarówno dobre rozwiązania, jak i błędy.

؞Vibecraftedٜ nie próbuje tego zmienić.

Zamiast tego:

- dopuszcza niedoskonałość pierwszego draftu
- wymusza iteracje
- zbiera kontrprzykłady
- zamyka pętlę dopiero wtedy, gdy system przestaje się psuć

To podejście jest bliższe metodom inżynieryjnym niż „promptowaniu”.

---

### Origin

؞Vibecraftedٜ nie powstał jako teoria.

Powstał z potrzeby dowiezienia realnego produktu przez osoby spoza tradycyjnej ścieżki software engineering. Praca z
agentami szybko pokazała ograniczenia: brak globalnego kontekstu, nadpisywanie działających fragmentów, rosnący chaos.

Z tej presji powstał zestaw zasad:

- najpierw zobacz system
- potem działaj
- potem pytaj „co jeszcze jest źle?”
- powtarzaj aż odpowiedź brzmi „nic”

Z czasem zasady stały się workflowami. Workflowy stały się frameworkiem.
