# VetCoders Ownership

Full-spectrum ownership mode for sessions where the user wants Codex to drive a
product slice from A to Z instead of waiting for stepwise instructions.

Use it when the brief sounds like:

- take ownership
- you drive
- od a do z
- wow effect
- make it feel finished
- dowiez to cale

## What It Covers

`vc-ownership` is broader than a coding helper. It is for:

- architecture decisions
- full-stack implementation
- runtime verification
- UX and visual polish
- docs and packaging
- agent orchestration
- desktop/browser reality checks when appropriate

## What Makes It Different

The skill assumes:

- the user wants fewer questions
- reasonable assumptions are good
- the repo is only part of the product
- wow effect comes from completeness, not decoration alone

## Relationship To Other VibeCraft Skills

This skill often composes with:

- `vc-partner` for shared executive debugging and framing
- `vc-agents` for external field-team agents
- `vc-marbles` for convergence loops
- `vc-dou` for product-surface gap finding
- `vc-decorate` and `vc-hydrate` for finish and market polish
